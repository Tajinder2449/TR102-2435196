### Learning Summary

#### Data Pipeline
A data pipeline is a series of processes that collect, transform, and move data from one system to another for analysis or machine learning. It ensures that data flows efficiently from its source to the final destination while maintaining data quality.

**Typical Stages of a Data Pipeline:**
- Data Collection
- Data Ingestion
- Data Cleaning
- Data Transformation
- Data Storage
- Data Analysis and Visualization

#### Data Cleaning
Data cleaning is the process of identifying and correcting inaccurate, incomplete, duplicate, or inconsistent data to improve its quality before analysis.

**Importance of Data Cleaning:**
- Improves data accuracy.
- Enhances model performance.
- Reduces errors during analysis.
- Ensures reliable decision-making.

#### Handling Missing Values
Missing values occur when some data entries are unavailable or blank.

**Common Techniques:**
- Removing rows or columns with excessive missing values.
- Replacing missing values with the mean, median, or mode.
- Using interpolation techniques.
- Filling values based on domain knowledge.

#### Handling Outliers
Outliers are data points that differ significantly from the rest of the dataset and may affect statistical analysis and machine learning models.

**Common Methods for Handling Outliers:**
- Detection using Box Plot or Interquartile Range (IQR).
- Detection using Z-Score.
- Removing extreme outliers.
- Capping or Winsorizing values.
- Transforming data using logarithmic or scaling techniques.
